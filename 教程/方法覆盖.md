# 方法覆盖

有时候你需要修改 SPT 服务器的原有行为。SPT 提供了两种方法覆盖机制：**继承覆盖**和 **Harmony 补丁**。

## 一、继承覆盖（推荐）

通过继承原有类并重写其方法来修改行为。

**源文件：** [6OverrideMethod/OverrideMethod.cs](../6OverrideMethod/OverrideMethod.cs)

### 基本原理

1. 找到要修改的类
2. 创建子类继承它
3. 重写要修改的方法
4. 使用相同的加载优先级替换原类

### 代码示例

```csharp
using SPTarkov.DI.Annotations;
using SPTarkov.Server.Core.DI;
using SPTarkov.Server.Core.Models.Utils;
using SPTarkov.Server.Core.Servers;
using SPTarkov.Server.Core.Services;

// 使用与原类相同的加载优先级
[Injectable(TypePriority = OnLoadOrder.Watermark)]
public class MyWatermark(
    ISptLogger<Watermark> logger,  // 注意：Logger 用原类的类型
    ConfigServer configServer,
    ServerLocalisationService localisationService,
    WatermarkLocale watermarkLocale
) : Watermark(logger, configServer, localisationService, watermarkLocale)  // 继承原类
{
    public override async Task OnLoad()
    {
        // 你的自定义逻辑
        logger.Success("这是自定义的水印覆盖！");
        
        // 调用原始方法（可选）
        await base.OnLoad();
    }
}
```

### 关键点

| 要点 | 说明 |
|------|------|
| 加载优先级 | 必须与原类相同，如 `OnLoadOrder.Watermark` |
| 构造函数参数 | 必须与原类一致 |
| Logger 类型 | 使用原类的类型，如 `ISptLogger<Watermark>` |
| base 调用 | 可选，`base.OnLoad()` 执行原始逻辑 |

## 二、Harmony 补丁（高级）

更强大的补丁机制，可以在任何方法的前后插入代码。

**源文件：** [6.1OverrideMethodHarmony/OverrideMethodHarmony.cs](../6.1OverrideMethodHarmony/OverrideMethodHarmony.cs)

### 基本原理

1. 创建补丁类继承 `AbstractPatch`
2. 指定要补丁的目标方法
3. 定义前缀/后缀方法
4. 在 OnLoad 中启用补丁

### 补丁类型

| 补丁类型 | 说明 |
|---------|------|
| `[PatchPrefix]` | 在原方法执行前运行 |
| `[PatchPostfix]` | 在原方法执行后运行 |
| `[PatchTranspiler]` | 直接修改方法的 IL 代码（高级） |

### 代码示例

```csharp
using System.Reflection;
using SPTarkov.DI.Annotations;
using SPTarkov.Reflection.Patching;
using SPTarkov.Server.Core.DI;
using SPTarkov.Server.Core.Models.Utils;
using SPTarkov.Server.Core.Utils;

// 补丁启用器
[Injectable(TypePriority = OnLoadOrder.PreSptModLoader)]
public class MyPatchEnabler(
    ISptLogger<MyPatchEnabler> logger) : IOnLoad
{
    public Task OnLoad()
    {
        // 启用补丁
        new MyCustomPatch().Enable();
        logger.Success("Harmony 补丁已加载！");
        return Task.CompletedTask;
    }
}

// 补丁定义
public class MyCustomPatch : AbstractPatch
{
    // 指定要补丁的方法
    protected override MethodBase GetTargetMethod()
    {
        return typeof(App).GetMethod(nameof(App.InitializeAsync));
    }

    // 前置补丁：在原方法执行前运行
    [PatchPrefix]
    public static bool Prefix()
    {
        // 这里可以执行自定义逻辑
        
        // 返回 true 继续执行原方法
        // 返回 false 跳过原方法
        return true;
    }

    // 后置补丁：在原方法执行后运行
    [PatchPostfix]
    public static async Task Postfix(Task __result)
    {
        // 异步方法需要 await 结果
        await __result;
        
        // 后续逻辑...
    }
}
```

### 访问原方法的参数和返回值

```csharp
public class ExamplePatch : AbstractPatch
{
    protected override MethodBase GetTargetMethod()
    {
        return typeof(SomeClass).GetMethod("SomeMethod");
    }

    [PatchPrefix]
    public static void Prefix(string param1, int param2)
    {
        // 可以直接使用原方法的参数名
        Console.WriteLine($"参数1: {param1}, 参数2: {param2}");
    }

    [PatchPostfix]
    public static void Postfix(string __result)
    {
        // __result 是原方法的返回值
        Console.WriteLine($"返回值: {__result}");
    }
}
```

## 两种方式对比

| 特性 | 继承覆盖 | Harmony 补丁 |
|------|---------|-------------|
| 复杂度 | 简单 | 较复杂 |
| 适用范围 | 只能覆盖虚方法 | 可以补丁任何方法 |
| 性能影响 | 小 | 略大 |
| 调试难度 | 容易 | 较难 |
| 推荐场景 | 优先使用 | 无法继承时使用 |

## 小贴士

1. **优先使用继承**：如果原方法是 virtual 的，用继承更简单
2. **注意命名空间**：确保引用了正确的类
3. **测试很重要**：覆盖方法可能影响其他功能，务必测试
4. **日志调试**：在补丁中加入日志，帮助排查问题
5. **查看源码**：了解原方法的实现，避免破坏其功能
