# 日志系统

日志是调试模组的重要工具。通过日志，你可以在服务器控制台和日志文件中输出信息，帮助了解模组的运行状态。

**源文件：** [1Logging/Logging.cs](../1Logging/Logging.cs)

## 为什么需要日志？

开发模组时，你经常需要知道：
- 模组是否成功加载？
- 某个变量的值是什么？
- 有没有出错？错在哪里？

日志就像是模组的"黑匣子"，记录运行过程中的重要信息。

## 如何使用日志

### 1. 注入日志服务

在你的类中，通过构造函数注入 `ISptLogger`：

```csharp
using SPTarkov.Server.Core.Models.Utils;

public class MyMod(
    ISptLogger<MyMod> logger  // 注意：尖括号里写你的类名
) : IOnLoad
{
    public Task OnLoad()
    {
        // 使用 logger 输出信息
        logger.Success("模组加载成功！");
        return Task.CompletedTask;
    }
}
```

### 2. 日志类型

日志分为几种类型，每种有不同的用途和显示效果：

```csharp
// 成功信息 - 绿色文字，表示操作成功
logger.Success("数据加载完成！");

// 警告信息 - 黄色文字，表示需要注意但不影响运行
logger.Warning("配置文件使用了默认值");

// 错误信息 - 红色文字，表示出现了问题
logger.Error("无法找到配置文件！");

// 普通信息 - 白色文字，一般性的提示
logger.Info("正在处理数据...");

// 严重错误 - 高亮红色，表示致命问题
logger.Critical("数据库连接失败！");

// 调试信息 - 只写入日志文件，不在控制台显示
logger.Debug("变量 x 的值是: " + x);
```

### 3. 彩色日志

如果你想让日志更醒目，可以自定义颜色：

```csharp
using SPTarkov.Server.Core.Models.Logging;

// 自定义文字颜色和背景颜色
logger.LogWithColor(
    "这是一条醒目的彩色日志！", 
    LogTextColor.Red,      // 红色文字
    LogBackgroundColor.Yellow  // 黄色背景
);
```

**可用的颜色：**

| 文字颜色 | 背景颜色 |
|---------|---------|
| `LogTextColor.Black` | `LogBackgroundColor.Black` |
| `LogTextColor.Red` | `LogBackgroundColor.Red` |
| `LogTextColor.Green` | `LogBackgroundColor.Green` |
| `LogTextColor.Yellow` | `LogBackgroundColor.Yellow` |
| `LogTextColor.Blue` | `LogBackgroundColor.Blue` |
| `LogTextColor.Magenta` | `LogBackgroundColor.Magenta` |
| `LogTextColor.Cyan` | `LogBackgroundColor.Cyan` |
| `LogTextColor.White` | `LogBackgroundColor.White` |

## 完整示例

```csharp
using SPTarkov.DI.Annotations;
using SPTarkov.Server.Core.DI;
using SPTarkov.Server.Core.Models.Logging;
using SPTarkov.Server.Core.Models.Spt.Mod;
using SPTarkov.Server.Core.Models.Utils;

namespace MyMod;

public record ModMetadata : AbstractModMetadata
{
    public override string ModGuid { get; init; } = "com.example.mymod";
    public override string Name { get; init; } = "日志示例";
    public override string Author { get; init; } = "开发者";
    public override SemanticVersioning.Version Version { get; init; } = new("1.0.0");
    public override SemanticVersioning.Range SptVersion { get; init; } = new("~4.0.0");
    public override string License { get; init; } = "MIT";
}

[Injectable(TypePriority = OnLoadOrder.PreSptModLoader + 1)]
public class MyMod(
    ISptLogger<MyMod> logger
) : IOnLoad
{
    public Task OnLoad()
    {
        // 各种日志类型示例
        logger.Success("这是成功信息");
        logger.Warning("这是警告信息");
        logger.Error("这是错误信息");
        logger.Info("这是普通信息");
        logger.Critical("这是严重错误");
        
        // 调试信息（只在日志文件中可见）
        logger.Debug("这是调试信息，不会显示在控制台");
        
        // 彩色日志
        logger.LogWithColor("这是彩色日志", LogTextColor.Red, LogBackgroundColor.Black);
        
        return Task.CompletedTask;
    }
}
```

## 日志文件位置

日志文件保存在 SPT 服务器的 `user/logs/` 目录下，按日期命名。调试时可以打开日志文件查看详细信息。

## 小贴士

1. **合理使用日志类型**：不要所有信息都用 `Success`，根据实际情况选择
2. **调试用 Debug**：开发阶段的调试信息用 `logger.Debug()`，不会污染控制台
3. **关键操作要记录**：重要的初始化、配置加载、错误处理都要有日志
4. **避免过多日志**：不要在循环中频繁输出日志，会影响性能
