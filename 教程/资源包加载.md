# 资源包加载

如果你想添加自定义的模型、贴图、音频等游戏资源，需要创建资源包模组。

**源文件：** [12Bundle/](../12Bundle/)

## 什么是 Bundle？

Bundle 是 Unity 的资源打包格式。SPT 使用 Bundle 来加载自定义的游戏资源，比如：
- 新武器模型
- 自定义贴图
- 音效文件

## 创建 Bundle 模组

### 1. 配置 ModMetadata

```csharp
public record ModMetadata : AbstractModMetadata
{
    public override string ModGuid { get; init; } = "com.example.bundle";
    public override string Name { get; init; } = "资源包示例";
    public override string Author { get; init; } = "开发者";
    public override SemanticVersioning.Version Version { get; init; } = new("1.0.0");
    public override SemanticVersioning.Range SptVersion { get; init; } = new("~4.0.0");
    public override bool? IsBundleMod { get; init; } = true;  // 标记为资源包模组
    public override string License { get; init; } = "MIT";
}
```

**关键点：** `IsBundleMod = true`

### 2. 创建 bundles.json

在模组根目录创建 `bundles.json`：

```json
{
    "manifest": [
        {
            "key": "assets/content/weapons/my_weapon/textures/client_assets.bundle",
            "dependencyKeys": []
        },
        {
            "key": "assets/content/weapons/my_weapon/models/client_assets.bundle",
            "dependencyKeys": [
                "assets/content/weapons/my_weapon/textures/client_assets.bundle"
            ]
        }
    ]
}
```

**字段说明：**
- `key`：Bundle 文件相对于 `bundles/` 目录的路径
- `dependencyKeys`：依赖的其他 Bundle（按顺序加载）

### 3. 放置 Bundle 文件

```
MyBundleMod/
├── bundles.json           # 清单文件
├── MyBundleMod.cs         # 模组代码
├── bundles/               # Bundle 文件目录
│   └── assets/
│       └── content/
│           └── weapons/
│               └── my_weapon/
│                   ├── textures/
│                   │   └── client_assets.bundle
│                   └── models/
│                       └── client_assets.bundle
```

### 4. 模组代码

```csharp
using SPTarkov.DI.Annotations;
using SPTarkov.Server.Core.DI;
using SPTarkov.Server.Core.Models.Spt.Mod;
using SPTarkov.Server.Core.Models.Utils;

namespace MyBundleMod;

public record ModMetadata : AbstractModMetadata
{
    public override string ModGuid { get; init; } = "com.example.bundle";
    public override string Name { get; init; } = "资源包示例";
    public override string Author { get; init; } = "开发者";
    public override SemanticVersioning.Version Version { get; init; } = new("1.0.0");
    public override SemanticVersioning.Range SptVersion { get; init; } = new("~4.0.0");
    public override bool? IsBundleMod { get; init; } = true;
    public override string License { get; init; } = "MIT";
}

[Injectable(TypePriority = OnLoadOrder.PostSptModLoader)]
public class BundleExample(
    ISptLogger<BundleExample> logger
) : IOnLoad
{
    public Task OnLoad()
    {
        logger.Success("资源包模组已加载！");
        return Task.CompletedTask;
    }
}
```

## 加载顺序

资源包模组通常使用 `OnLoadOrder.PostSptModLoader`，确保在 SPT 核心加载完成后再加载。

## 依赖关系

如果 Bundle A 依赖 Bundle B，需要在 `dependencyKeys` 中声明：

```json
{
    "manifest": [
        {
            "key": "shared_assets.bundle",
            "dependencyKeys": []
        },
        {
            "key": "weapon_textures.bundle",
            "dependencyKeys": ["shared_assets.bundle"]
        },
        {
            "key": "weapon_model.bundle",
            "dependencyKeys": ["shared_assets.bundle", "weapon_textures.bundle"]
        }
    ]
}
```

## 创建 Unity Bundle

Bundle 文件需要使用 Unity 编辑器创建：

1. 在 Unity 中导入资源
2. 设置 AssetBundle 名称
3. 使用 BuildPipeline 构建 AssetBundle

这部分涉及到 Unity 开发，不在本教程范围内。

## 小贴士

1. **必须设置 IsBundleMod**：否则 Bundle 不会被加载
2. **路径要正确**：Bundle 文件路径必须和 bundles.json 中的 key 匹配
3. **依赖顺序**：确保依赖的 Bundle 先加载
4. **文件大小**：大 Bundle 会增加加载时间，考虑拆分
5. **测试验证**：进入游戏检查资源是否正确加载
