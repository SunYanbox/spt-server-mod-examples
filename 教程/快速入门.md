# 快速入门 - 模组基础结构

每个 SPT 服务器模组都必须包含一个 **ModMetadata** 类来定义模组的基本信息。这是模组的"身份证"，服务器启动时会读取它。

## 为什么需要 ModMetadata？

想象一下，如果每个模组都没有标识信息，服务器怎么知道：
- 这个模组叫什么名字？
- 是谁开发的？
- 需要什么版本的 SPT 才能运行？
- 和其他模组会不会冲突？

ModMetadata 就是用来回答这些问题的。

## 完整代码示例

```csharp
using SPTarkov.Server.Core.Models.Spt.Mod;

public record ModMetadata : AbstractModMetadata
{
    // 唯一标识符 - 推荐使用反向域名格式
    public override string ModGuid { get; init; } = "com.example.mymod";
    
    // 模组名称 - 显示给玩家看的
    public override string Name { get; init; } = "我的第一个模组";
    
    // 作者名称
    public override string Author { get; init; } = "你的名字";
    
    // 协作者列表（可选）
    public override List<string>? Contributors { get; init; } = ["帮忙的朋友"];
    
    // 模组版本号 - 遵循语义化版本规范
    public override SemanticVersioning.Version Version { get; init; } = new("1.0.0");
    
    // 支持的 SPT 版本范围
    public override SemanticVersioning.Range SptVersion { get; init; } = new("~4.0.0");
    
    // 不兼容的模组列表（可选）
    public override List<string>? Incompatibilities { get; init; } = ["冲突模组的ModGuid"];
    
    // 依赖的其他模组（可选）
    public override Dictionary<string, SemanticVersioning.Range>? ModDependencies { get; init; }
    
    // 项目主页（可选）
    public override string? Url { get; init; } = "https://github.com/你的项目";
    
    // 是否是资源包模组（添加模型、贴图等）
    public override bool? IsBundleMod { get; init; } = false;
    
    // 开源许可证
    public override string License { get; init; } = "MIT";
}
```

## 属性详解

| 属性 | 必需 | 说明 |
|------|------|------|
| `ModGuid` | 是 | 模组唯一ID，建议用反向域名格式如 `com.example.mymod` |
| `Name` | 是 | 模组显示名称 |
| `Author` | 是 | 作者名字 |
| `Version` | 是 | 模组版本，如 `1.0.0` |
| `SptVersion` | 是 | 支持的 SPT 版本，`~4.0.0` 表示兼容 4.0.x |
| `License` | 是 | 许可证类型，如 `MIT`、`Apache-2.0` |
| `Contributors` | 否 | 协作者列表 |
| `Incompatibilities` | 否 | 冲突模组的 ModGuid 列表 |
| `ModDependencies` | 否 | 依赖模组及其版本要求 |
| `Url` | 否 | 项目主页地址 |
| `IsBundleMod` | 否 | 是否包含 Unity 资源包 |

## 版本号说明

### 语义化版本（Semantic Versioning）

格式：`主版本号.次版本号.修订号`

- **主版本号**：不兼容的重大改动
- **次版本号**：向后兼容的新功能
- **修订号**：向后兼容的问题修复

### 版本范围语法

| 写法 | 含义 |
|------|------|
| `~4.0.0` | 兼容 4.0.x（如 4.0.1、4.0.5） |
| `^4.0.0` | 兼容 4.x.x（如 4.1.0、4.2.0） |
| `>=4.0.0` | 4.0.0 及以上版本 |

## 最小化示例

如果你只需要最基础的配置：

```csharp
public record ModMetadata : AbstractModMetadata
{
    public override string ModGuid { get; init; } = "com.example.simplemod";
    public override string Name { get; init; } = "简单模组";
    public override string Author { get; init; } = "开发者";
    public override SemanticVersioning.Version Version { get; init; } = new("1.0.0");
    public override SemanticVersioning.Range SptVersion { get; init; } = new("~4.0.0");
    public override string License { get; init; } = "MIT";
}
```

## 小贴士

1. **ModGuid 要唯一**：避免和别人的模组冲突，推荐用你的域名或 GitHub 用户名
2. **版本号要规范**：方便玩家和服务器判断是否需要更新
3. **声明冲突**：如果已知和某些模组不兼容，一定要写出来，避免玩家踩坑
