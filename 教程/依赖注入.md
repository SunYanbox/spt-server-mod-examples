# 依赖注入

依赖注入（DI）是 SPT 架构的核心。理解它能帮助你写出更好的模组代码。

**源文件：** [11RegisterClassesInDI/RegisterClassesInDI.cs](../11RegisterClassesInDI/RegisterClassesInDI.cs)

## 什么是依赖注入？

传统方式是你自己创建对象：

```csharp
// 传统方式（不推荐）
public class MyMod
{
    private Logger _logger = new Logger();
    private Database _database = new Database();
}
```

依赖注入是告诉框架"我需要什么"，框架自动提供：

```csharp
// 依赖注入方式（推荐）
public class MyMod(
    ISptLogger<MyMod> logger,    // 框架自动注入
    DatabaseService database      // 框架自动注入
)
{
    // 直接使用，不需要自己创建
}
```

## Injectable 属性

用 `[Injectable]` 标记的类会被自动注册：

```csharp
using SPTarkov.DI.Annotations;

// 默认是 Scoped 生命周期
[Injectable]
public class MyService { }

// 单例模式
[Injectable(InjectionType.Singleton)]
public class MySingleton { }

// 带加载优先级
[Injectable(TypePriority = OnLoadOrder.PostDBModLoader + 1)]
public class MyMod : IOnLoad { }
```

## 生命周期

### Singleton（单例）

整个应用只有一个实例：

```csharp
[Injectable(InjectionType.Singleton)]
public class CounterService
{
    private int _count = 0;
    
    public void Increment() => _count++;
    public int GetCount() => _count;
}
```

**特点：**
- 全局唯一
- 状态持久化
- 适合：配置、缓存、共享服务

### Scoped（作用域）

每次使用时创建新实例：

```csharp
[Injectable]  // 或 [Injectable(InjectionType.Scoped)]
public class TempDataService
{
    private int _value = 0;
    
    public void SetValue(int v) => _value = v;
    public int GetValue() => _value;
}
```

**特点：**
- 每次注入可能是新实例
- 状态独立
- 适合：请求处理、临时数据

## 实际对比

```csharp
// 单例示例：计数器会累加
[Injectable(InjectionType.Singleton)]
public class SingletonCounter(ISptLogger<SingletonCounter> logger)
{
    private int _count = 0;
    
    public void IncrementAndLog()
    {
        _count++;
        logger.Success($"单例计数: {_count}");  // 输出: 1, 2, 3...
    }
}

// 作用域示例：计数器不会累加
[Injectable(InjectionType.Scoped)]
public class ScopedCounter(ISptLogger<ScopedCounter> logger)
{
    private int _count = 0;
    
    public void IncrementAndLog()
    {
        _count++;
        logger.Success($"作用域计数: {_count}");  // 输出: 1, 1, 1...
    }
}
```

## 完整示例

```csharp
using SPTarkov.DI.Annotations;
using SPTarkov.Server.Core.DI;
using SPTarkov.Server.Core.Models.Spt.Mod;
using SPTarkov.Server.Core.Models.Utils;

namespace MyMod;

public record ModMetadata : AbstractModMetadata
{
    public override string ModGuid { get; init; } = "com.example.di";
    public override string Name { get; init; } = "依赖注入示例";
    public override string Author { get; init; } = "开发者";
    public override SemanticVersioning.Version Version { get; init; } = new("1.0.0");
    public override SemanticVersioning.Range SptVersion { get; init; } = new("~4.0.0");
    public override string License { get; init; } = "MIT";
}

// 单例服务：全局配置
[Injectable(InjectionType.Singleton)]
public class ConfigService
{
    public string ServerName { get; set; } = "我的服务器";
    public int MaxPlayers { get; set; } = 10;
}

// 作用域服务：临时数据
[Injectable(InjectionType.Scoped)]
public class RequestContext
{
    public string CurrentPlayer { get; set; }
}

// 主模组类
[Injectable(TypePriority = OnLoadOrder.PostDBModLoader + 1)]
public class MyMod(
    ISptLogger<MyMod> logger,
    ConfigService config,      // 注入单例
    RequestContext context     // 注入作用域
) : IOnLoad
{
    public Task OnLoad()
    {
        // 使用注入的服务
        logger.Success($"服务器名称: {config.ServerName}");
        logger.Info($"最大玩家数: {config.MaxPlayers}");
        
        return Task.CompletedTask;
    }
}
```

## 构造函数注入详解

```csharp
[Injectable]
public class MyComplexService(
    ISptLogger<MyComplexService> logger,    // 日志
    DatabaseService database,                // 数据库
    ConfigServer configServer,               // 配置
    AnotherService another                   // 其他服务
)
{
    // 所有依赖都已自动注入，直接使用
}
```

## 何时用 Singleton vs Scoped

| 场景 | 推荐 |
|------|------|
| 配置管理 | Singleton |
| 缓存服务 | Singleton |
| 全局状态 | Singleton |
| 请求处理 | Scoped |
| 临时数据 | Scoped |
| 无状态计算 | 都可以 |

## 小贴士

1. **默认用 Scoped**：除非确实需要全局状态，否则用默认的 Scoped
2. **避免循环依赖**：A 依赖 B，B 又依赖 A 会导致启动失败
3. **接口优先**：可以注入接口而不是具体类，更灵活
4. **构造函数简洁**：不要注入太多依赖，考虑拆分类
5. **理解生命周期**：选择错误的生命周期可能导致状态混乱
