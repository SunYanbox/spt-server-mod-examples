# 多类协作

当模组功能变复杂时，把所有代码写在一个类里会变得难以维护。SPT 的依赖注入系统让你可以轻松地将代码拆分到多个类中。

**源文件：** [7UseMultipleClasses/](../7UseMultipleClasses/)

## 为什么需要多类？

想象一个添加商人的模组，它需要：
- 读取配置文件
- 创建商人数据
- 添加商品
- 设置本地化文本
- ...

如果全部写在一个类里，代码会很长很乱。拆分成多个类可以让代码更清晰。

## 核心概念

### Injectable 属性

用 `[Injectable]` 标记的类会被自动注册到依赖注入容器：

```csharp
using SPTarkov.DI.Annotations;
using SPTarkov.Server.Core.DI;

[Injectable]  // 默认是 Scoped 生命周期
public class MyHelperClass
{
    public string GetMessage() => "来自 Helper 的消息";
}

[Injectable(InjectionType.Singleton)]  // 单例模式
public class MyCacheService
{
    private readonly Dictionary<string, object> _cache = new();
    
    public void Add(string key, object value) => _cache[key] = value;
    public object Get(string key) => _cache.GetValueOrDefault(key);
}
```

### 构造函数注入

在构造函数参数中声明需要的类，框架会自动注入：

```csharp
[Injectable(TypePriority = OnLoadOrder.PostDBModLoader + 1)]
public class MainClass(
    ISptLogger<MainClass> logger,
    MyHelperClass helper,        // 自动注入
    MyCacheService cache         // 自动注入
) : IOnLoad
{
    public Task OnLoad()
    {
        var message = helper.GetMessage();
        logger.Info(message);
        return Task.CompletedTask;
    }
}
```

## 完整示例

### 主类：UseMultipleClasses.cs

```csharp
using SPTarkov.DI.Annotations;
using SPTarkov.Server.Core.DI;
using SPTarkov.Server.Core.Models.Spt.Mod;
using SPTarkov.Server.Core.Models.Utils;

namespace MyMod;

public record ModMetadata : AbstractModMetadata
{
    public override string ModGuid { get; init; } = "com.example.multipleclasses";
    public override string Name { get; init; } = "多类协作示例";
    public override string Author { get; init; } = "开发者";
    public override SemanticVersioning.Version Version { get; init; } = new("1.0.0");
    public override SemanticVersioning.Range SptVersion { get; init; } = new("~4.0.0");
    public override string License { get; init; } = "MIT";
}

[Injectable(TypePriority = OnLoadOrder.PostDBModLoader + 1)]
public class MainClass(
    ISptLogger<MainClass> logger,
    HelperClass helper
) : IOnLoad
{
    public Task OnLoad()
    {
        // 调用其他类的方法
        var result = helper.ProcessData("测试数据");
        logger.Success($"处理结果: {result}");
        return Task.CompletedTask;
    }
}
```

### 辅助类：HelperClass.cs

```csharp
using SPTarkov.DI.Annotations;
using SPTarkov.Server.Core.DI;
using SPTarkov.Server.Core.Models.Utils;

namespace MyMod;

[Injectable]  // 自动注册到 DI 容器
public class HelperClass(
    ISptLogger<HelperClass> logger  // 也可以注入其他服务
)
{
    public string ProcessData(string input)
    {
        logger.Info($"正在处理: {input}");
        return $"已处理: {input.ToUpper()}";
    }
}
```

## 生命周期

### Singleton（单例）

整个应用只有一个实例：

```csharp
[Injectable(InjectionType.Singleton)]
public class GlobalConfig
{
    public string Setting { get; set; } = "默认值";
}
```

适合：
- 配置管理
- 缓存服务
- 共享状态

### Scoped（作用域）

每次请求创建新实例（默认）：

```csharp
[Injectable]  // 或 [Injectable(InjectionType.Scoped)]
public class RequestHandler
{
    private int _count = 0;
    
    public void Increment() => _count++;
}
```

适合：
- 请求处理
- 临时数据
- 无状态服务

## 多类协作的实际应用

### 示例：商人模组的类划分

```
MyTraderMod/
├── ModMetadata.cs        # 模组元数据
├── TraderLoader.cs       # 主加载逻辑
├── TraderConfigReader.cs # 读取配置
├── AssortBuilder.cs      # 构建商品列表
├── LocaleHelper.cs       # 本地化处理
└── DialogHelper.cs       # 对话处理
```

各司其职，代码清晰。

## 小贴士

1. **单一职责**：每个类只做一件事
2. **命名清晰**：类名要能体现其功能
3. **合理注入**：不要过度依赖注入，简单逻辑可以放一起
4. **注意循环依赖**：A 依赖 B，B 又依赖 A 会导致错误
5. **文件组织**：可以每个类一个文件，方便管理
